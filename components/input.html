Input

Text input field for forms and data entry. Includes validation states, helper text, and various input types.

Overview

Input components allow users to enter text, numbers, and other data. They are fundamental form elements that should provide clear labels, validation feedback, and accessibility features.

Usage Guidelines

Do

  • Always provide clear labels
  • Use appropriate input types
  • Provide helpful validation messages
  • Show success states when appropriate
  • Use placeholder text as examples, not labels

Don't

  • Don't use placeholder as a label replacement
  • Don't validate before user has finished typing
  • Don't use vague error messages
  • Don't make required fields unclear
  • Don't disable without explaining why

Code Example

import { Input } from '@albi/flight';

// Basic input
<Input
  label="Email Address"
  type="email"
  placeholder="user@example.com"
  required
/>

// Input with error
<Input
  label="Password"
  type="password"
  error="Password must be at least 8 characters"
/>

// Input with prefix and suffix
<Input
  label="Website"
  type="url"
  prefix="https://"
  suffix={<LinkIcon />}
/>
