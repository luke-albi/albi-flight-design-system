Button

Primary interactive element for user actions. Supports multiple variants, sizes, and states.

View in Figma

Overview

Buttons are interactive elements that users can click or tap to trigger actions. They should be used for primary actions like submitting forms, confirming operations, or navigating to important pages.

Variants

The Button component comes in multiple visual variants to indicate hierarchy and importance:

Primary

Main call-to-action

Secondary

Secondary actions

Outline

Tertiary actions

Ghost

Minimal emphasis

Danger

Destructive actions

States

Buttons support multiple interactive states:

Default

Hover

Active

Focused

Disabled

Loading

Props

Prop Type Default Description
variant "primary" | "secondary" | "outline" | "ghost" | "danger" "primary" Visual style variant
size "sm" | "md" | "lg" "md" Button size
disabled boolean false Disable button interaction
loading boolean false Show loading state
fullWidth boolean false Make button full width
icon ReactNode undefined Optional icon element
onClick (event: MouseEvent) => void undefined Click handler

Usage Guidelines

Do

  • Use for primary actions users should take
  • Limit to 1-2 primary buttons per screen
  • Use clear, action-oriented labels
  • Ensure adequate touch target size (44x44px minimum)
  • Provide visual feedback on interaction

Don't

  • Don't use multiple primary buttons together
  • Don't use for navigation (use links instead)
  • Don't use vague labels like "Click here"
  • Don't make buttons too small to tap
  • Don't use disabled state without explanation

Accessibility

  • Keyboard Navigation: Fully accessible via Tab and Enter/Space keys
  • Screen Readers: Use aria-label when button text isn't descriptive
  • Focus Indication: Clear focus ring for keyboard users
  • Loading State: Announce loading state with aria-busy
  • Disabled State: Use aria-disabled and provide context

Code Example

import { Button } from '@albi/flight';

// Primary button
<Button variant="primary" onClick={handleSubmit}>
  Submit Form
</Button>

// Button with loading state
<Button
  variant="primary"
  loading={isLoading}
  disabled={isLoading}
  onClick={handleSave}
>
  Save Changes
</Button>

// Button with icon
<Button variant="secondary" icon={<PlusIcon />}>
  Add Item
</Button>

Platform-Specific Notes

Web

  • Supports hover states and cursor changes
  • Can be used as a link with href prop
  • Supports target and rel attributes for links

Mobile

  • Uses haptic feedback on press
  • Larger default touch targets (48x48dp)
  • No hover state (uses press state instead)
