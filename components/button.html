Button

Primary interactive element for user actions. Supports multiple styles (Filled, Outlined, Text), color variants, and interaction states.

Overview

Buttons are interactive elements that users can click or tap to trigger actions. The Albi Flight button system provides three distinct styles and four color variants, each with complete interaction states including enabled, hovered, focused, pressed, and disabled.

Usage Guidelines

Do

  • Use Filled style for primary actions
  • Use Outlined style for secondary actions
  • Use Text style for tertiary or inline actions
  • Limit to 1-2 primary buttons per screen
  • Use clear, action-oriented labels
  • Maintain consistent color meaning (Danger for destructive)

Don't

  • Don't use multiple Filled Primary buttons together
  • Don't mix button styles randomly without hierarchy
  • Don't use for navigation (use links instead)
  • Don't use vague labels like "Click here"
  • Don't make buttons too small to tap (min 48px)
  • Don't use disabled state without explanation

Code Example

import { Button } from '@albi/flight';

// Filled Primary Button (default)
<Button>
  Submit Form
</Button>

// Outlined Secondary Button
<Button style="outlined" color="secondary">
  Cancel
</Button>

// Text Danger Button
<Button style="text" color="danger">
  Delete
</Button>

// Button with loading state
<Button
  style="filled"
  color="primary"
  loading={isLoading}
  disabled={isLoading}
  onClick={handleSave}
>
  Save Changes
</Button>

// Button with icon
<Button style="outlined" icon={<PlusIcon />}>
  Add Item
</Button>

Accessibility

  • Keyboard Navigation: Fully accessible via Tab and Enter/Space keys
  • Focus Indicators: 16% opacity state layer on focus
  • Screen Readers: Use aria-label when button text isn't descriptive
  • Disabled State: Use aria-disabled and provide context why button is disabled
  • Loading State: Announce loading state with aria-busy="true"
  • Color Contrast: All color combinations meet WCAG AA standards

Platform-Specific Notes

Web

  • Supports hover states with 8% opacity state layer
  • Can be used as a link with href prop
  • Cursor changes to pointer on hover

Mobile

  • Uses haptic feedback on press
  • Larger minimum touch targets (48x48dp)
  • No hover state (uses pressed state instead)
  • Optimized tap animations with state layers
